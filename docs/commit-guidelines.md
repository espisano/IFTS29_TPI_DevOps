# Commit Guidelines (breve)

Se recomiendan los siguientes prefijos en los mensajes de commit:

- `feat:` — nueva funcionalidad
- `fix:` — corrección de un bug
- `docs:` — cambios en documentación
- `chore:` — tareas de mantenimiento/infraestructura
- `test:` — añadir o modificar tests

Ejemplo de mensaje:

```
docs: add CONTRIBUTING.md with contribution flow
```

Mantener los mensajes cortos y claros; en el cuerpo del commit explicar cambios complejos.
