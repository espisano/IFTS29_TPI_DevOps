# Contributing

Gracias por contribuir a este proyecto. Sugerencias para colaborar sin romper la aplicación:

- Branches: crear una rama con el prefijo `feature/` o `chore/`, por ejemplo `feature/tu-nombre-docs`.
- Commits: usar prefijos en mensajes: `feat:`, `fix:`, `chore:`, `docs:`, `test:`.
- Pull Requests: abrir PR contra `develop` o `main` (según flujo del equipo). Incluir una descripción breve.
- Tests: si agregas tests, procura que sean pequeños y no dependan de datos reales; usa mocks.
- Secrets: **NO** subir archivos con credenciales. Usa `.env.example` para documentar variables.

Flujo rápido:
1. Crear rama: `git checkout -b feature/tu-nombre-descripcion`
2. Hacer cambios pequeños y claros.
3. `git add` y `git commit -m "docs: add ..."`
4. Push y abrir PR.

Gracias — mantengamos el repo ordenado y seguro.
